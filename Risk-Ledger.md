## 一、合规风险

### 1.1 政治敏感内容红线

| 序号 | 禁忌类型 | 具体描述 | 来源/案例 |
|-----|---------|---------|----------|
| 1 | 历史虚无主义 | 否定中国革命历史、中国共产党领导的正确性、社会主义制度优越性 | 《互联网文化管理暂行规定》 |
| 2 | 政治讽刺 | 影射、讽刺国家领导人或现行政策 | 起点内容审核指南 |
| 3 | 分裂国家 | 涉及台湾、西藏、新疆、香港等地区主权问题的不当表述 | 《网络出版服务管理规定》 |
| 4 | 敏感历史事件 | 对抗日战争、解放战争等重大历史事件的歪曲描述 | 各平台审核标准 |
| 5 | 境外势力美化 | 美化西方政治制度或丑化中国政治制度 | 案例：某平台下架多部"恨国"作品 |

**具体禁止表述**：
- 不得将近代史中侵略者描写为正面人物
- 不得对红军、八路军、解放军进行负面描写
- 历史穿越作品不得改变重大历史结局的政治正确性
- 架空历史不得明显影射现实政治

**违规处罚标准**：
| 违规等级 | 触发条件 | 处罚措施 |
|---------|---------|---------|
| 轻微违规 | 首次触碰边缘内容 | 警告 + 要求修改章节 |
| 一般违规 | 重复违规或单次明确违规 | 屏蔽章节 + 24小时禁止更新 |
| 严重违规 | 触碰政治敏感红线 | 作品下架 + 账号封禁 |
| 极端违规 | 严重违反法律法规 | 永久封禁 + 可能追究法律责任 |

## 五、熔断机制 [S01-QA-038]

### 5.1 概述

熔断机制是项目的"紧急制动系统"。当风险达到特定阈值时，必须停止更新进行修复，避免问题扩大化。本章节定义三级预警机制：黄色预警、红色预警、黑色预警。

---

### 5.2 黄色预警（一级警报）

**定义**：项目出现异常信号，需要引起重视并主动干预，但不影响正常更新。

#### 触发条件

| 编号 | 触发条件 | 数据阈值 |
|-----|---------|---------|
| Y01 | 订阅连续下跌 | 连续3章订阅下跌>10% |
| Y02 | 追读率下降 | 周追读率下降>15% |
| Y03 | 差评激增 | 单章差评数>正常值3倍 |
| Y04 | 存稿预警 | 存稿数量<10章 |
| Y05 | 读者质疑 | 同类型质疑评论>5条/章 |
| Y06 | 爽点缺失 | 连续3章无中型以上爽点 |
| Y07 | 节奏拖沓 | 连续3章无实质剧情推进 |

#### 应对动作

| 动作编号 | 应对措施 | 责任人 | 时限 |
|---------|---------|-------|-----|
| YA01 | **召开专家会诊** | PM + 全体专家 | 24小时内 |
| YA02 | **数据诊断** | MA（市场分析） | 分析数据下跌根因 |
| YA03 | **内容复盘** | CA（首席架构） | 检查近期剧情问题 |
| YA04 | **制定调整方案** | PM（项目经理） | 48小时内出方案 |
| YA05 | **执行调整** | 相关专家 | 按方案执行 |
| YA06 | **效果跟踪** | PM（项目经理） | 1周内复查 |

#### 会诊流程

```
触发黄色预警 → 24小时内召开专家会诊 → 诊断问题根因 → 制定调整方案 → 执行调整 → 1周后复查
```

**专家会诊议程**：
1. 数据呈现：展示触发预警的具体数据
2. 根因分析：讨论问题可能的原因
3. 方案讨论：提出2-3个解决方案
4. 方案决策：选定最优方案
5. 责任分配：明确执行责任人
6. 时限确定：设定完成deadline

#### 解除条件

| 条件 | 说明 |
|-----|------|
| 数据恢复 | 触发指标回归正常范围 |
| 问题解决 | 根因已被解决 |
| 持续观察 | 连续7天无新问题 |

---

### 5.3 红色预警（二级警报）

**定义**：项目出现严重问题，必须暂停更新进行紧急修复。

#### 触发条件

| 编号 | 触发条件 | 数据阈值/说明 |
|-----|---------|-------------|
| R01 | 重大设定矛盾 | 主角能力/规则体系出现不可调和的逻辑矛盾 |
| R02 | 严重人物OOC | 主角行为严重偏离人设，读者大量投诉 |
| R03 | 数据断崖 | 单日订阅下跌>30% |
| R04 | 存稿耗尽 | 存稿数量<3章 |
| R05 | 平台警告 | 收到平台内容审核警告 |
| R06 | 差评风暴 | 单章差评数>正常值10倍，或多章持续差评 |
| R07 | 黄色预警升级 | 黄色预警触发后1周内未改善 |

#### 应对动作

| 动作编号 | 应对措施 | 责任人 | 时限 |
|---------|---------|-------|-----|
| RA01 | **立即暂停更新** | PM | 即时 |
| RA02 | **发布暂停公告** | PM + MS | 1小时内 |
| RA03 | **召开紧急会议** | 全体专家 | 4小时内 |
| RA04 | **问题定位** | CA + LK | 明确矛盾/问题所在 |
| RA05 | **制定修复方案** | CA + PM | 24小时内 |
| RA06 | **执行修复** | 相关专家 | 按方案执行 |
| RA07 | **修复验证** | QA | 确认修复效果 |
| RA08 | **恢复更新** | PM | 修复完成后 |

#### 暂停公告模板

```
【紧急公告】

各位书友：

由于[原因简述]，《诡仙界》暂停更新[X]天进行调整。

预计恢复时间：[日期]

感谢大家的理解与支持，调整期间我会全力优化内容，回来后会有[福利/加更]回馈大家。

——林守一（作者）
```

#### 修复流程

```
触发红色预警 → 立即暂停更新 → 发布公告 → 4小时内召开紧急会议 → 24小时内定方案 → 执行修复 → 验证通过 → 恢复更新
```

#### 解除条件

| 条件 | 说明 |
|-----|------|
| 问题修复 | 矛盾/问题已被修复 |
| 方案验证 | QA确认修复方案有效 |
| 存稿恢复 | 存稿恢复至5章以上 |
| 平台解禁 | 平台警告已解除 |

### 5.4 黑色预警（三级警报）

**定义**：项目遭遇重大危机，必须全面停更进行根本性整改。

#### 触发条件

| 编号 | 触发条件 | 说明 |
|-----|---------|-----|
| B01 | 触碰平台红线 | 作品被屏蔽/下架/账号被警告 |
| B02 | 设定体系崩溃 | 核心设定（规则免疫/诡仙界规则）出现系统性矛盾，无法简单修复 |
| B03 | 口碑全面崩坏 | 大规模读者弃书，评论区全面差评 |
| B04 | 红色预警升级 | 红色预警触发后修复失败 |
| B05 | 法律/版权风险 | 涉嫌抄袭、侵权等法律风险 |

#### 应对动作

| 动作编号 | 应对措施 | 责任人 | 时限 |
|---------|---------|-------|-----|
| BA01 | **全面停更** | PM | 即时 |
| BA02 | **发布停更公告** | PM + MS | 1小时内 |
| BA03 | **与平台沟通** | PM | 如涉及平台问题 |
| BA04 | **全文审查** | 全体专家 | 全面梳理问题 |
| BA05 | **根因分析** | CA + PM | 找出系统性问题 |
| BA06 | **制定重大整改方案** | 全体专家 | 3天内 |
| BA07 | **用户确认** | 用户 | 重大方案需用户批准 |
| BA08 | **执行整改** | 相关专家 | 按方案执行 |
| BA09 | **整改验收** | QA + 用户 | 确认整改效果 |
| BA10 | **恢复更新决策** | 用户 | 决定是否恢复 |

#### 全面审查清单

| 审查维度 | 审查内容 |
|---------|---------|
| **合规审查** | 全文搜索敏感词，检查政治/色情/暴力内容 |
| **设定审查** | 比对设定库，检查所有设定矛盾 |
| **人物审查** | 检查主角/配角是否OOC |
| **剧情审查** | 检查伏笔回收、因果逻辑 |
| **质量审查** | 检查错别字、文风一致性 |

#### 停更公告模板

```
【重要公告】

各位书友：

非常抱歉地通知大家，由于[原因简述]，《诡仙界》需要进行一次全面整改。

在整改期间，更新将暂停。

我深知这会影响到大家的阅读体验，但这是为了给大家呈现更好的故事。我会利用这段时间：
1. [整改内容1]
2. [整改内容2]
3. [整改内容3]

预计整改周期：[X]天/周

感谢大家一路以来的支持。整改完成后，我会以更好的状态回归。

——林守一（作者）
```

#### 解除条件

| 条件 | 说明 |
|-----|------|
| 整改完成 | 所有整改项目已完成 |
| 验收通过 | QA + 用户确认整改效果 |
| 平台解禁 | 如涉及平台问题，需平台解禁 |
| 用户批准 | 用户同意恢复更新 |

### 5.5 预警升级机制

| 当前等级 | 升级条件 | 升级后等级 |
|---------|---------|-----------|
| 正常 | 触发黄色预警条件 | 黄色预警 |
| 黄色预警 | 1周内未改善，或触发红色预警条件 | 红色预警 |
| 红色预警 | 修复失败，或触发黑色预警条件 | 黑色预警 |

---

### 5.6 预警降级机制

| 当前等级 | 降级条件 | 降级后等级 |
|---------|---------|-----------|
| 黄色预警 | 连续7天指标正常 | 正常 |
| 红色预警 | 问题修复+连续3天稳定 | 黄色预警 |
| 黑色预警 | 整改完成+验收通过+用户批准 | 正常 |

---

### 5.7 熔断机制汇总表

| 预警等级 | 颜色 | 严重程度 | 更新状态 | 响应时间 | 决策权限 |
|---------|-----|---------|---------|---------|---------|
| **一级** | 🟡 黄色 | 异常信号 | 正常更新 | 24小时内 | AI自主 |
| **二级** | 🔴 红色 | 严重问题 | 暂停更新 | 4小时内 | AI建议+用户确认 |
| **三级** | ⚫ 黑色 | 重大危机 | 全面停更 | 即时 | 用户批准 |

---

### 5.8 熔断机制执行原则

| 原则 | 说明 |
|-----|------|
| **早发现早处理** | 问题发现越早，修复成本越低 |
| **宁可误报不可漏报** | 宁可触发一次虚惊，不可放过真正的风险 |
| **透明沟通** | 遇到问题及时与读者沟通，诚实面对 |
| **快速响应** | 预警触发后必须在规定时间内响应 |
| **彻底修复** | 不做表面文章，找到根因彻底解决 |
| **复盘总结** | 每次预警解除后进行复盘，避免重蹈覆辙 |
